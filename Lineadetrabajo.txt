1) Introducción y objetivos
Introducción

Construiremos una eCommerce “tipo marketplace” (inspirada en Olist) con un dashboard de métricas alimentado por datos históricos reales: ~100k órdenes (2016–2018) con información de clientes, vendedores, pagos, entregas y reseñas. 
ngocyen99tb.lelouvincx.com

Objetivos

Importar y normalizar los CSV de Olist en SQLite.

Exponer REST APIs para:

Catálogo (productos/categorías),

Ventas (órdenes/ítems/pagos),

Operación (entregas/SLAs),

Experiencia (reviews),

Analytics (KPIs y rankings).

Front en React + Vite + Redux Toolkit con:

vista “Tienda demo” (catálogo y detalle),

panel “Analytics” con filtros por fechas/estado/categoría/estado (UF).

Mantener diseño escalable con dominios y arquitectura C4.

2) Ventaja competitiva y valor añadido

Dataset real y relacional: no es un CSV “plano”; permite trazabilidad de punta a punta (venta → pago → entrega → review). 
ngocyen99tb.lelouvincx.com

Analítica de logística: puedes medir “entregado vs estimado” usando order_delivered_customer_date y order_estimated_delivery_date. 
ngocyen99tb.lelouvincx.com

Marketplace-ready: Olist trae sellers, así que puedes mostrar métricas por vendedor (muy atractivo para un portfolio). 
ngocyen99tb.lelouvincx.com

Segmentación de clientes: trae customer_unique_id para agrupar compras del mismo cliente a lo largo del tiempo. 
ngocyen99tb.lelouvincx.com

3) Funcionalidades clave (basadas en Olist)
Catálogo (read-only / demo)

Listado de productos con categoría (PT + traducción EN)

Filtros: categoría, rango de precio (derivado), “top vendidos”

Detalle: dimensiones/peso/fotos_qty

Olist no trae “nombre comercial” del producto, trae atributos (category, weight, etc.). 
ngocyen99tb.lelouvincx.com

Para UI: puedes mostrar product_id como SKU y la categoría como nombre visible.

Ventas / Órdenes

Listado de órdenes con estado y timestamps del ciclo de vida (compra, aprobación, despacho, entrega, estimado). 
ngocyen99tb.lelouvincx.com

Detalle de orden: ítems (price, freight_value) y vendedor por ítem. 
ngocyen99tb.lelouvincx.com

Pagos: múltiples pagos por orden (payment_sequential, payment_type, payment_value). 
ngocyen99tb.lelouvincx.com

Reviews / Satisfacción

Score 1–5 + texto opcional + fecha. 
ngocyen99tb.lelouvincx.com

Geo / Distribución

Clientes y sellers por estado/ciudad/ZIP prefix

Mapa (opcional): geolocation tiene múltiples filas por ZIP (hay que agregar/deduplicar). 
ngocyen99tb.lelouvincx.com

Analytics (KPIs)

Ventas (GMV) por día/semana/mes

Órdenes totales, ítems vendidos

AOV (ticket promedio)

Top categorías, top sellers, top productos

Entregas: % entregadas tarde vs estimado, lead time promedio

Reviews: score promedio por categoría/seller/tiempo

Definiciones sugeridas (claras y consistentes)

GMV: SUM(order_items.price) (sin flete) o SUM(price + freight_value) (con flete). Olist trae ambos a nivel ítem. 
ngocyen99tb.lelouvincx.com

Pagado: SUM(order_payments.payment_value) (ojo: puede haber split payments). 
ngocyen99tb.lelouvincx.com

4) LeanCanva (Lean Canvas)

Problema

No hay visibilidad integral (ventas + logística + satisfacción) sin herramientas externas.

Segmentos

Analistas eCommerce / operación logística / sellers en marketplaces.

Propuesta de valor

Dashboard que cruza ventas + entrega + reviews en un mismo modelo de datos.

Solución

React dashboard + Django APIs + dataset Olist importado.

Canales

Demo online, GitHub, portfolio, LinkedIn.

Métricas clave

GMV, órdenes, AOV, lateness rate, lead time, review score.

Ventaja injusta

Dataset real multi-tabla y arquitectura por dominios lista para crecer.

Analista revisa “late deliveries” comparando entregado vs estimado.

7) Arquitectura del sistema
Backend (Django + DRF + SQLite)

Enfoque recomendado (2 capas de datos):

Staging Olist (raw): tablas “tal cual CSV” para trazabilidad y re-importación.

Core dominio (opcional): vistas/materializaciones para simplificar métricas (ej. sales_daily_agg).

Componentes:

ingestion: comandos manage.py import_olist_*

analytics: endpoints agregados (KPIs)

catalog: productos/categorías (con traducción)

sales: orders/items/payments

reviews: order_reviews

geo: geolocation + agregación por ZIP

Frontend (React + Vite + Redux Toolkit)

Rutas:

/catalog

/orders (listado analítico / admin)

/analytics (dashboard)

Redux slices:

filtersSlice (fechas, estados, categoría, seller, estado BR)

kpisSlice

timeseriesSlice

rankingsSlice (top products/sellers/categories)

logisticsSlice (late rate, lead time)

reviewsSlice (scores)
